# Project Rules — cloudflare-heimdall

1. **Terraform Cloud, not CLI** — This workspace (`infrastructure` org: `ahlooii`) is VCS-driven. Pushing changes to `terraform/cloudflare-heimdall/` on the `main` branch triggers an automatic plan/apply in Terraform Cloud. Do not run `terraform` CLI locally unless you first run `terraform login`.

2. **Cloudflare API key lives in Terraform Cloud** — The `CLOUDFLARE_API_KEY` and `CLOUDFLARE_EMAIL` are configured as workspace variables in TFC. They are not in any local file.

3. **Tunnel architecture** — A single Cloudflare named tunnel (`oracle_tunnel`) routes traffic to the Oracle x64 instance:
   - `g.ahlooii.com` → `ssh://localhost:2279` (SSH via Cloudflare Access + service token)
   - `git.ahlooii.com` → `http://localhost:3300` (Gitea, temporary until ARM recovery)

4. **Security hardening applied** — Geoblock WAF (SG/MY only) with skip rule for tunnel hostname. Cloudflare Access with non-expiring service token on SSH. Ports 80/443 closed on Oracle VCN — all traffic goes through the tunnel. Do not re-open direct ports without explicit approval.

5. **Sensitive outputs** — `tunnel_token`, `access_client_id`, `access_client_secret` are marked sensitive. View them in TFC States tab → latest state → raw JSON. Never commit or log their values.

6. **File naming convention** — `_main.tf` and `_variables.tf` use underscore prefix for provider/backend/variable definitions. `heimdall.tf` contains all resource definitions.
